# ブログシステム 要件定義

## 1. システム概要
ASP.NET Core MVCをベースとしたブログシステムを構築する。
個人または小規模チームでのブログ運営を想定した、シンプルで使いやすいシステムを目指す。

## 2. 機能要件

### 2.1 記事管理機能
- **記事一覧表示**
  - ページング対応（1ページあたり10件）
  - 公開日時の降順で表示
  - タイトル、概要、投稿日時、カテゴリ、タグを表示

- **記事詳細表示**
  - 記事の全文表示
  - Markdown対応
  - カテゴリ・タグの表示
  - 前の記事・次の記事へのナビゲーション

- **記事作成・編集**
  - タイトル、本文（Markdown）、概要の入力
  - カテゴリの選択（単一）
  - タグの設定（複数）
  - 公開/下書き状態の管理
  - 公開日時の設定
  - アイキャッチ画像のアップロード

- **記事削除**
  - 論理削除による記事削除
  - 削除確認ダイアログ表示

### 2.2 カテゴリ管理機能
- カテゴリの作成・編集・削除
- カテゴリ別記事一覧表示
- カテゴリ名、スラッグ、説明の管理

### 2.3 タグ管理機能
- タグの作成・編集・削除
- タグ別記事一覧表示
- タグクラウド表示

### 2.4 コメント機能
- 記事へのコメント投稿
- コメントの表示（階層なし）
- コメントの承認制（管理者による承認）
- コメントの削除（管理者）

### 2.5 認証・認可機能
- **管理者ログイン**
  - ASP.NET Core Identityによる認証
  - ユーザー名・パスワードによるログイン
  - ログアウト

- **権限管理**
  - 管理者：すべての機能にアクセス可能
  - 一般ユーザー：記事閲覧とコメント投稿のみ

### 2.6 検索機能
- キーワードによる記事検索
- タイトル・本文を対象とした全文検索

### 2.7 管理画面
- ダッシュボード（記事数、コメント数などの統計表示）
- 記事管理画面
- カテゴリ管理画面
- タグ管理画面
- コメント管理画面

## 3. 非機能要件

### 3.1 性能要件
- ページ表示速度：3秒以内
- 同時アクセス：100ユーザーまで対応

### 3.2 セキュリティ要件
- CSRF対策（ASP.NET Core標準機能）
- XSS対策（適切なエスケープ処理）
- SQLインジェクション対策（Entity Framework Coreのパラメータ化クエリ）
- パスワードのハッシュ化（ASP.NET Core Identity）

### 3.3 ユーザビリティ要件
- レスポンシブデザイン対応（モバイル・タブレット・PC）
- 直感的な管理画面UI
- Markdownプレビュー機能

### 3.4 保守性要件
- コードの可読性を重視
- MVCパターンに従った設計
- 適切なコメント・ドキュメント

## 4. 画面一覧

### フロントエンド
1. トップページ（記事一覧）
2. 記事詳細ページ
3. カテゴリ別記事一覧ページ
4. タグ別記事一覧ページ
5. 検索結果ページ

### 管理画面
1. ログインページ
2. ダッシュボード
3. 記事一覧・作成・編集ページ
4. カテゴリ管理ページ
5. タグ管理ページ
6. コメント管理ページ

## 5. データ要件

### 5.1 データ保持期間
- 記事データ：無期限
- コメントデータ：無期限
- ログデータ：3ヶ月

### 5.2 バックアップ
- データベースの定期バックアップ（日次）
